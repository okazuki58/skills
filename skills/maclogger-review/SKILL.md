# Maclogger 振り返り

目標達成状況に焦点を当てた簡潔な振り返りを生成。週次目標と比較し、次週のアクションを明確化。約30-50行のマークダウンとして `/Users/ogawakazuki/dev/xincere-review/reviews/` に出力。

## ⚠️ 重要: 環境設定

**このスキルは作者の環境に特化しています。** 使用前に以下のパスを自分の環境に合わせて変更してください：

1. このファイル（`~/.claude/skills/maclogger-review/SKILL.md`）を開く
2. 以下の出力先パスを検索して変更:
   - `/Users/ogawakazuki/dev/xincere-review/reviews/` → あなたの振り返り保存先
   - `/Users/ogawakazuki/dev/maclogger/reports/` → あなたのログ保存先

または、以下の手順で初回セットアップを行ってください：

1. 振り返りファイルの保存先ディレクトリを作成:
   ```bash
   mkdir -p ~/reviews/weekly ~/reviews/monthly
   ```

2. このSKILL.mdファイル内の全てのパスを置換:
   - `/Users/ogawakazuki/dev/xincere-review/reviews/` → `~/reviews/`
   - `/Users/ogawakazuki/dev/maclogger/reports/` → （あなたのログディレクトリ、または削除）

## 振り返りモード

### モード1: 自動振り返り

週次目標と自動比較し、簡潔な振り返りを生成。

### モード2: 対話的壁打ちモード

以下の場合にAskUserQuestionで対話的な振り返りを実施:
- 「うまくいかなかったことを整理したい」
- 「今週の振り返りを対話的に進めたい」

**プロセス**:
1. 「今週はどうでしたか？」と質問
2. AskUserQuestionでうまくいったこと/いかなかったことを確認
3. 問題に対して「なぜ」で深掘り
4. 次週のアクションを対話的に決定
5. 振り返りファイルに保存

## ワークフロー

### モード1: 自動振り返り

1. **期間の特定**: ユーザーの期間を解釈（先週、今月、2025H2など）
2. **週次目標の読み込み**: `goals/YYYY-HX-週次目標.md` から目標を取得
3. **ログの読み取り**（オプション）: 必要に応じてmacloggerログを読み込み
4. **達成判定**: 各目標の達成/未達成を判定
5. **理由の抽出**: 未達成の簡潔な理由を抽出
6. **次週アクションの提案**: 未達成タスクの対応方法を提案
7. **ファイル出力**: 振り返りを生成・保存（約30-50行）
8. **ユーザーへ報告**: 出力パスとサマリーを報告

**重要**: 日次詳細は不要。週次目標との比較のみに焦点を当てる。

### モード2: 対話的壁打ち

1. **期間の特定**: どの期間を振り返るか（デフォルト: 先週）
2. **週次目標の読み込み**: `goals/YYYY-HX-週次目標.md` から取得
3. **自由な振り返り**: 「今週はどうでしたか？」
4. **AskUserQuestionで深掘り**:
   - Q1: 「今週うまくいったことは何ですか？」（複数選択: 週次目標項目 + その他）
   - Q2: 「今週うまくいかなかったことは何ですか？」（複数選択）
   - Q3: 「うまくいかなかった理由は何だと思いますか？」（各項目について）
   - Q4: 「なぜその理由が生まれたのでしょうか？」（深掘り）
   - Q5: 「次週はどうしますか？」（各未達成項目: 継続/中止/延期/アプローチ変更）
5. **次のアクションを決定**: 対話的に決定
6. **ファイル出力**: 生成・保存
7. **ユーザーへ報告**

## 振り返りファイルフォーマット

**目標**: 振り返り全体を約30-50行に（目標達成状況 + 次のアクションのみ）

**必須事項**:
1. マークダウンチェックボックス形式を使用: `[x]` 達成、`[ ]` 未達成 → 1行の理由を追加
2. すべて箇条書きベース（最小限の文章）
3. 週次目標との比較が最優先
4. 日次詳細は不要（「12/23 XXX実装、12/24 YYY追加」など）
5. Value別のサブセクションは不要（週次目標を直接リスト）
6. 未達成の理由: 簡潔（1行） - 「時間不足」「優先度変更」など
7. 次週への引き継ぎを明確化（すべて `[ ]` 形式）
8. 気づき/改善点: 2-3項目のみ

**フォーマット詳細**: [format-examples.md](references/format-examples.md)

## 重要なルール

1. **期間の明確化**: ユーザーが「先週」「今月」と言った場合、具体的な日付範囲を確認
2. **包括的なログ読み取り**: 期間のすべてのログを読む（漏れなし）
3. **事実ベース**: ログから事実のみを抽出（推測しない）
4. **正確なValue分類**: 最も適切なValueに分類
5. **正確な目標達成判定**: 目標ファイルと比較し、達成/未達成を正確に判定
6. **ファイル上書き確認**: 同じファイル名が存在する場合、上書きを確認
