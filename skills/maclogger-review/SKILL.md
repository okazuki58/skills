# maclogger自動振り返りスキル

## 概要

週次目標ファイルと照合し、**目標達成状況を中心とした簡潔な振り返り**を自動生成します。日次の作業詳細は記載せず、「何ができて何ができなかったか」「次週は何をすべきか」のみに焦点を当てます。出力先は`/Users/ogawakazuki/dev/xincere-review/reviews/`で、30-50行程度のmdファイルです。

## 使用タイミング

- 週次振り返りを作成したい時（毎週日曜日）
- 月次振り返りを作成したい時（月末）
- 半期振り返りを作成したい時（2月末、8月末）
- 目標達成状況を確認したい時

## 振り返りの2つのモード

### モード1: 自動振り返りモード（従来通り）

週次目標との照合を自動化し、簡潔な振り返りファイルを生成します。

### モード2: 壁打ち形式モード（NEW）

AskUserQuestionを使って対話的に振り返りを進めます。以下のような時に使用：
- 「うまくいかなかったことを整理したい」
- 「今週の振り返りを対話的に進めたい」
- 「次週に向けて何をすべきか一緒に考えたい」

**壁打ち形式のプロセス**：
1. 「今週はどうでしたか？」と自由に話してもらう
2. AskUserQuestionで「うまくいったこと」「うまくいかなかったこと」を質問
3. うまくいかなかったことについて「なぜ」を深掘り
4. 次週への改善アクションを対話的に決める
5. 振り返りファイルに保存

## 主要機能

### 1. 期間指定での振り返り生成

ユーザーが指定した期間のログを読み込み、振り返りを生成します。

**対応期間**:
- **週次**: 先週（月〜日）、今週（月〜今日）
- **月次**: 今月、先月、特定月（例：9月）
- **半期**: 今半期、前半期、特定半期（例：2025H2）

### 2. 週次目標との照合

週次目標ファイル（`goals/2025-H2-週次目標.md`）から該当週の目標を読み込み、以下の3つのValueごとに達成状況を判定：

#### Be hungry（貪欲になれ）
- 技術学習、記事執筆、アウトプットなどの目標

#### Be an owner（当事者たれ）
- タスク完了、チーム貢献、問題解決などの目標

#### Be sustainable（継続的であれ）
- ドキュメント化、自動化、効率化などの目標

各目標について、Markdownタスクチェック形式で記載：
- `[x]` 達成
- `[ ]` 未達成 → 理由を1行で記載

### 3. 振り返りファイルの出力

`/Users/ogawakazuki/dev/xincere-review/reviews/`に以下の形式でファイル出力：

- **週次**: `YYYY-WXX.md`（例：2025-W01.md）
- **月次**: `YYYY-MM.md`（例：2025-09.md）
- **半期**: `YYYY-HX.md`（例：2025-H2.md）

   選択肢:
   - [ ] 開発タスクが予想以上に時間がかかった
   - [ ] 計画が甘かった

   - [ ] 延期する
   - [ ] アプローチを変える
   User: 継続する（月・火午前に時間を確保）

   - うまくいかなかったこと: 記事執筆・AIドキュメント執筆
   - 理由: 時間を確保していなかった

## 振り返りファイルのフォーマット

**重要**: 週次目標の達成状況を中心にまとめる（目安：30-50行）

### 原則
1. **目標達成状況が最重要** - 計画していた目標が達成できたか/できなかったか
2. **タスクチェック形式を使用** - `[x]` 完了、`[ ]` 未完了
3. **全て箇条書きベース** - 簡潔に
4. **日次の詳細は不要** - 作業の詳細な列挙はしない
5. **次週へのアクションを明確に** - 未達成の目標を次週どうするか

```markdown
# YYYY-WXX 週次振り返り

**期間**: YYYY年MM月DD日（月）〜 YYYY年MM月DD日（日）

## 週次目標の達成状況

### Be hungry
- [x] 記事執筆：「XXX」を公開

### Be an owner
- [x] XXX機能の実装完了

### Be sustainable

### 新規タスク
- [ ] 記事2本目の執筆開始
- [ ] XXXの設計レビュー実施

## 気づき・改善点

- XXXに時間がかかりすぎた → 次週はYYYで効率化
- 計画より〇〇が遅れた → △△を調整する必要がある
```

## 実行フロー

### モード1: 自動振り返りモード

1. **期間の確認**: ユーザーが指定した期間を解釈（先週、今月、2025H2など）
2. **週次目標の読み込み**: `goals/2025-H2-週次目標.md` から該当週の目標を取得
3. **ログの読み込み**: macloggerから該当期間のログを読み込み（必要に応じて）
4. **目標達成状況の判定**: 各目標が達成できたか/できなかったかを判定
5. **未達成理由の抽出**: ログから未達成の理由を簡潔に抽出
6. **次週アクションの提案**: 未達成タスクをどうするか提案
7. **ファイル出力**: 振り返りファイルを生成・保存（30-50行程度）
8. **ユーザーに報告**: 出力先とサマリーを報告

**重要**: 日次の作業詳細は不要。週次目標との照合のみに集中する。

### モード2: 壁打ち形式モード

1. **期間の確認**: どの期間の振り返りをするか確認（デフォルト: 先週）
2. **週次目標の読み込み**: `goals/2025-H2-週次目標.md` から該当週の目標を取得
3. **自由な振り返り開始**: 「今週はどうでしたか？」と自由に話してもらう
4. **AskUserQuestionで深掘り**:
   - Q1: 「今週うまくいったことは何ですか？」（複数選択可）
     - 選択肢: 週次目標の項目 + その他
   - Q2: 「今週うまくいかなかったことは何ですか？」（複数選択可）
     - 選択肢: 週次目標の未達成項目 + その他
   - Q3: 「うまくいかなかった理由は何だと思いますか？」（各項目ごと）
     - 選択肢: 時間不足 / 優先度が低かった / やり方がわからなかった / その他
   - Q4: 「なぜその理由が生まれたのでしょうか？」（深掘り）
   - Q5: 「次週はどうしますか？」（各未達成項目ごと）
     - 選択肢: 継続する / 中止する / 延期する / アプローチを変える
5. **次週アクションの決定**: 対話的に次週のアクションを決める
6. **ファイル出力**: 振り返りファイルを生成・保存
7. **ユーザーに報告**: 出力先とサマリーを報告

## Claude Codeへの指示

振り返り生成時は、以下の手順を必ず実行すること：

### 必須ルール

1. **期間の明確化**: ユーザーが「先週」「今月」などと言った場合、具体的な日付範囲を確認
2. **ログの網羅的な読み込み**: 該当期間のログを全て読み込む（漏れがないように）
3. **事実ベースの記載**: ログに記載されている事実のみを抽出（推測や創作をしない）
4. **Value分類の精度**: 実績が複数のValueに該当する場合、最も適切なValueに分類
5. **目標達成状況の正確な判定**: 目標ファイルと照合し、達成/未達成を正確に判定
6. **ファイル上書き確認**: 既に同じファイル名の振り返りが存在する場合、上書きするか確認

### 目標達成状況中心の振り返りのための厳守事項

**目標**: 振り返り全体を30-50行程度にまとめる（目標達成状況と次週アクションのみ）

1. **Markdownタスクチェック形式を必ず使用**
   - 達成した目標：`[x]` 
   - 未達成の目標：`[ ]` → 理由を1行で追記
   - 次週のタスク：全て `[ ]` で記載

2. **全て箇条書きベースで記載**
   - 文章での説明は最小限
   - 箇条書きで簡潔に

3. **週次目標との照合が最優先**
   - `goals/2025-H2-週次目標.md` から該当週の目標を読み込む
   - 各目標が達成できたか/できなかったかを明確に判定

4. **日次の作業詳細は書かない**
   - 「12/23にXXXを実装、12/24にYYYを追加」のような日次ログは不要
   - 「XXX機能を実装」という結果のみを記載
   - 作業の詳細なプロセスは省略

5. **Value別のサブセクションは作らない**
   - 「技術学習」「アウトプット」「タスク完了」などのサブセクションは不要
   - 週次目標を直接列挙するだけ

6. **未達成の理由は簡潔に（1行）**
   - 「時間不足」「優先度変更」「技術的な困難」など、端的な理由のみ
   - 詳細な説明は不要

7. **次週への引き継ぎを明確に**
   - 未達成タスクを次週どうするか（継続/中止/延期）
   - 新規タスクがあれば追加
   - 全て `[ ]` 形式で記載

8. **気づき・改善点は2-3項目のみ**
   - 次週の改善につながる気づきのみ
   - 冗長な総評は不要
