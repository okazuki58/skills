## 使用例

### 例1: 週次振り返りの生成（自動モード）

```
ユーザー: 先週の振り返りを作成して
```

→ Claude:
1. `goals/2025-H2-週次目標.md`から先週（W52）の目標を読み込み
2. 各目標の達成/未達成を判定
3. 未達成の理由を簡潔に抽出
4. 次週への引き継ぎを明確化
5. `/Users/ogawakazuki/dev/xincere-review/reviews/weekly/2025-W52.md`に出力（30-50行）

### 例2: 壁打ち形式での振り返り

```
ユーザー: 今週うまくいかなかったことがあるので、一緒に振り返りたい
```

→ Claude:
1. 「今週はどうでしたか？自由に話してください。」
   User: 「今週は記事を書く予定だったけど、全然進まなかった。他の開発タスクが忙しくて…」

2. AskUserQuestion: 「今週うまくいかなかったことは何ですか？」
   選択肢:
   - [ ] 記事8本目の執筆・投稿
   - [ ] AIドキュメントChapter 3執筆
   - [ ] その他
   User: 1, 2 を選択

3. AskUserQuestion: 「記事が書けなかった理由は何だと思いますか？」
   選択肢:
   - [ ] 時間不足
   - [ ] 優先度が低かった
   - [ ] やり方がわからなかった
   - [ ] モチベーションが上がらなかった
   - [ ] その他
   User: 時間不足

4. AskUserQuestion: 「なぜ時間不足になったのでしょうか？」
   選択肢:
   - [ ] 開発タスクが予想以上に時間がかかった
   - [ ] 計画が甘かった
   - [ ] 記事執筆の時間を確保していなかった
   - [ ] その他
   User: 記事執筆の時間を確保していなかった

5. AskUserQuestion: 「次週はどうしますか？」
   選択肢:
   - [ ] 継続する（月・火午前に時間を確保）
   - [ ] 中止する
   - [ ] 延期する
   - [ ] アプローチを変える
   User: 継続する（月・火午前に時間を確保）

6. 振り返りファイルを生成・保存
   - うまくいかなかったこと: 記事執筆・AIドキュメント執筆
   - 理由: 時間を確保していなかった
   - 次週のアクション: 月・火午前に明示的に時間を確保する

### 月次振り返りの生成

ユーザー: 12月の振り返りを作成して
