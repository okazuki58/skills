## 目標記載フォーマット

**重要**: 各目標には必ず対応する評価項目を明記すること

### フォーマット例

**重要**: 全ての目標に `[ ]` をつけて、後から振り返りでチェックできるようにする

```markdown
## Be hungry（貪欲になれ）

### 【評価項目1: 貪欲に新しいものを勉強したか】

- [ ] 設計関連の技術書を2冊以上読破（DB設計、アーキテクチャパターンなど）
- [ ] 学んだ設計パターンを実プロジェクトで3回以上適用
- [ ] AWSの体系的な学習を行う（AWS認定資格取得を目指す）

### 【評価項目2: 貪欲に仕事に取り組んだか】

- [ ] 技術ブログ/Qiita記事を隔週で投稿 → **半期で12本以上**
- [ ] 詳細な技術ドキュメントを2本作成（Confluence/Google Slide/動画解説など）
- [ ] AIを使った設計の最適化手法をドキュメント化

## Be an owner（当事者たれ）

### 【評価項目1: 締め切り意識を持てたか】

- [ ] すべてのタスクを期限内に完了（やむを得ない場合は事前相談）
- [ ] 週次報告を欠かさず実施

### 【評価項目2: チーム全体の生産性に寄与したか】

- [ ] 月1回のチーム勉強会を主催
- [ ] コードレビューを積極的に実施（週5件以上）
- [ ] 新メンバーのオンボーディング支援

## Be sustainable（継続的であれ）

### 【評価項目1: 属人化しないように仕組み化するようになにかやれたか】

- [ ] 開発手順書を5本以上作成
- [ ] 既存の属人化タスクを2つ以上ドキュメント化・引き継ぎ

### 【評価項目2: 日常業務に対して改善する動きができたか】

- [ ] CI/CDパイプラインの改善（ビルド時間20%短縮）
- [ ] 繰り返し作業の自動化スクリプトを3つ以上作成
```
