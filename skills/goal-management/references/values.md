# Value評価基準

## 組織の共通価値観（行動指針）

### Be hungry（貪欲になれ）

**理念**：結果にこだわり、もっと高みを目指し続ける（そのために学び続ける）組織でありたい。常に現状に満足せず、貪欲に行動せよ。

**評価項目**：
1. 貪欲に新しいものを勉強したか
2. 貪欲に仕事に取り組んだか

**具体例（高評価）**：
- 新しい技術・ツールを積極的に学習し、業務に活用した
- 業務範囲外でも学習を続け、チームに知見を共有した
- 現状に満足せず、より高い目標に挑戦した
- 困難なタスクに対しても諦めず、粘り強く取り組んだ

**具体例（低評価）**：
- 最低限のタスクをこなすだけで、学習意欲が見られない
- 新しいことへの挑戦を避け、慣れた作業のみを選んでいる

### Be an owner（当事者たれ）

**理念**：組織として結果を出すために、その中での"当事者"としてなにができるのか、自分がリーダーならどう動くのかということを常に考え行動する。その事業・組織における責任感を持ち、やると言ったことは遂行する。

**評価項目**：
1. 締め切り意識を持てたか
2. チーム全体の生産性に寄与したか

**具体例（高評価）**：
- デッドラインを意識し、計画的にタスクを完遂した
- 自分のタスクだけでなく、チーム全体の進捗を気にかけた
- 他メンバーのサポートを積極的に行った
- 問題が発生した際、他責にせず自分ができることを考えて行動した
- コミットしたことは必ず実行し、できない場合は早めに相談した

**具体例（低評価）**：
- 締め切りを守れず、理由も報告しない
- 自分のタスクだけに集中し、チーム全体を見ていない
- やると言ったことを実行せず、フォローもない

### Be sustainable（継続的であれ）

**理念**：同じ作業を永遠と繰り返したり、属人的な仕事の仕方は継続性がないため効率化や仕組み化の意識を持つ。また、健康にも注意を払い常に高いパフォーマンスを発揮できるように仕事をすること。

**評価項目**：
1. 属人化しないように仕組み化するようになにかやれたか
2. 日常業務に対して改善する動きができたか

**具体例（高評価）**：
- マニュアル・ドキュメント化を進め、誰でもできる状態にした
- 繰り返し作業を自動化・効率化した
- チーム内で知識共有の仕組みを作った
- 業務プロセスの改善提案を行い、実行した
- 健康管理に気を配り、安定したパフォーマンスを維持した

**具体例（低評価）**：
- 同じ作業を手作業で繰り返している
- 自分しか知らない・できない状態を放置している
- 改善の余地があっても現状維持を選んでいる
- 無理な働き方でパフォーマンスが低下している

## 評価ランク

| ランク | 意味 | 給与への影響 |
|--------|------|------------|
| S | 想定を大きく超えている | 給与アップ |
| A | 期待値ちょい上 | 給与アップ |
| B | 期待値通り | 給与ステイ |
| C | 期待値に達していない | 前回C以下の場合、給与ダウン |
| D | 出来ているとはいえない | 給与ダウン |

## 評価プロセス

### 1. 被評価者による自己評価

事実ベースでアピールポイントを記載する。

**重要**：必ず具体的な事実・エピソードを記載すること
- [NG] 悪い例：「新しい技術を勉強しました」
- [OK] 良い例：「TypeScript 5.0の新機能を学習し、チーム勉強会で共有。プロジェクトに導入し、型安全性が向上しました」

### 2. 事実の正当性チェック

評価者が事実の正しさを評価：

| 評価 | 意味 |
|------|------|
| S | 事実と評価が一致していると思う |
| A | 事実と評価が少し乖離している |
| B | 事実と評価が著しく乖離している |

Bが発生した場合、評価者が上位グレードの評価者と相談し、評価を決定する。

### 3. バリュー評価

事実評価がA・Sの場合、評価者が評価理由を添えて最終評価（S/A/B/C/D）を記載。

### 4. ランダムチェック

評価者と同等のグレードを持った役職者によるランダムな正当性チェックを行い、評価の偏りを防ぐ。
