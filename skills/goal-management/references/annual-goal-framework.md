# 年間目標設定フレームワーク

## 概要

半期目標の前に、より大きな視点で1年間の目標を設定します。年間目標を設定することで、各半期の目標が一貫性を持ち、長期的なキャリア成長に繋がります。

## 年間目標の設定タイミング

以下のいずれかのタイミングで設定します：

1. **年度初め（4月）**: 会社の年度に合わせて4月に次の3月までの1年間目標を設定
2. **暦年初め（1月）**: 1月に次の12月までの1年間目標を設定
3. **半期ごとに見直し**: 3月・9月に次の1年間を見据えた長期目標を設定し、その半期分に分解

## 年間目標の構成

年間目標は以下の要素で構成されます：

### 1. 年間テーマ（1つ）

今年1年で達成したい大きなテーマを設定します。

**例**：
- 「フルスタックエンジニアとして自走できるようになる」
- 「テックリードとしてチームをリードできる技術力を身につける」
- 「G3昇進を目指し、自発的にタスクを切り出して進められるようになる」

### 2. Value別の年間目標（3つ）

3つのValue（Be hungry / Be an owner / Be sustainable）ごとに、1年間で達成したい具体的な目標を設定します。

#### Be hungry（貪欲になれ）

**目標の視点**：
- 新しい技術・知識を習得する
- アウトプット・発信を継続する
- 現状に満足せず、高みを目指す

**年間目標の例**：
- 技術書を6冊読破し、学んだことを実務で活用する
- 技術ブログを月2本ペースで年間24本投稿する
- AWS認定資格を2つ取得する

#### Be an owner（当事者たれ）

**目標の視点**：
- 締め切りを守り、約束を果たす
- チーム全体の生産性に貢献する
- 当事者意識を持って行動する

**年間目標の例**：
- 全てのタスクを期限内に完了（遅延0件）
- コードレビューを週10件以上実施し、チームの品質向上に貢献
- 新メンバーのメンター役を務め、オンボーディングを成功させる

#### Be sustainable（継続的であれ）

**目標の視点**：
- 属人化を解消し、仕組み化する
- 日常業務を改善・効率化する
- 継続可能な働き方を実現する

**年間目標の例**：
- 開発手順書を12本作成（月1本ペース）
- CI/CDパイプラインを改善し、ビルド時間を30%短縮
- 自動化スクリプトを6本作成し、繰り返し作業を削減

### 3. 成果指標（KPI）

年間目標の達成度を測るための具体的な指標を設定します。

**例**：
- 技術書読破数: 6冊
- ブログ投稿数: 24本
- コードレビュー数: 480件（週10件×48週）
- 開発手順書作成数: 12本

## 年間目標を半期目標に分解する方法

年間目標を2つの半期目標に分解します。

### 分解の原則

1. **前半期（H1: 3-8月 or 上半期）**: 基礎固め・習得フェーズ
   - 新しいスキルの学習
   - 基本的なパターンの理解
   - 小規模な実践

2. **後半期（H2: 9-2月 or 下半期）**: 応用・定着フェーズ
   - 学んだスキルの実践
   - 大規模な適用
   - チームへの展開

### 分解の例

**年間目標**：
```
技術書を6冊読破し、学んだことを実務で活用する
```

**半期目標への分解**：
- **H1（3-8月）**: 技術書3冊を読破し、基本パターンを理解・小規模な実践をする
- **H2（9-2月）**: 技術書3冊を読破し、学んだパターンを大規模プロジェクトに適用する

---

**年間目標**：
```
技術ブログを月2本ペースで年間24本投稿する
```

**半期目標への分解**：
- **H1（3-8月）**: 技術ブログを12本投稿（月2本ペース）
- **H2（9-2月）**: 技術ブログを12本投稿（月2本ペース）

---

**年間目標**：
```
CI/CDパイプラインを改善し、ビルド時間を30%短縮
```

**半期目標への分解**：
- **H1（3-8月）**: CI/CDパイプラインのボトルネック分析と改善案の策定
- **H2（9-2月）**: 改善案の実装とビルド時間30%短縮の達成

## 年間目標の見直しタイミング

年間目標は固定ではなく、状況に応じて見直します：

- **半期末（2月・8月）**: 前半期の振り返りと後半期の調整
- **四半期ごと**: 進捗確認と軌道修正
- **大きな変化があった時**: 役割変更、プロジェクト変更、グレード変更時

## ファイル保存形式

年間目標は以下の形式で保存します：

```
goals/
├── YYYY-年間目標.md          # 年間目標（例: 2025-年間目標.md）
├── YYYY-H1-半期目標.md       # 前半期目標
├── YYYY-H2-半期目標.md       # 後半期目標
├── YYYY-H1-月次目標.md
├── YYYY-H2-月次目標.md
├── YYYY-H1-週次目標.md
└── YYYY-H2-週次目標.md
```
