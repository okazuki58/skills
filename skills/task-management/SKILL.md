# タスク管理

対話的計画立案、体系的な分解、自動化されたGitHub Projects/Issuesセットアップを通じて、GitHubリポジトリベースのタスク管理をサポート。

## 前提条件

このスキルは **GitHub CLI (`gh`)** と **GitHubリポジトリ** が必要です。

```bash
# GitHub CLIのインストール（macOS）
brew install gh

# 認証
gh auth login
```

現在のディレクトリがGitHubリポジトリでない場合、またはGitHub CLIが利用できない場合は、タスク分解と計画立案のみを提供できます。

## ワークフロー

1. リポジトリ分析
2. 対話的計画立案（AskUserQuestion）
3. タスク分解（大→中→小）
4. GitHub Projects作成
5. Issue作成（優先度、ラベル、マイルストーン）
6. 依存関係セットアップ
7. 進捗管理開始

## ステップ1: リポジトリ分析

現在の状態を理解するために分析:
- README.md - 概要、ステータス、技術スタック
- ディレクトリ構造 - フロントエンド/バックエンド/インフラ
- 既存のIssue - オープン/クローズ、ラベル
- コミット履歴 - 最近のアクティビティ、頻度

ユーザーにサマリーを提示。

## ステップ2: 対話的計画立案

AskUserQuestionで以下を確認:

**Q1: 目標** - 新機能 / 改善 / バグ修正 / リファクタリング / パフォーマンス / ドキュメント / テスト

**Q2: 優先度** - 選択した目標をランク付け

**Q3: タイムライン** - 1週間 / 2週間 / 1ヶ月 / 2-3ヶ月 / 期限なし

**Q4: チームサイズ** - 1人 / 2-3人 / 4-5人 / 6人以上

**Q5: 粒度** - 大（1-2週間）/ 中（2-3日）/ 小（半日〜1日）タスクあたり

## ステップ3: タスク分解

目標を実行可能なタスクに分解:

1. **大タスク**（3-5個）- 主要マイルストーン
2. **中タスク**（大タスクあたり3-5個）- 機能コンポーネント
3. **小タスク** - 個別のIssue（1タスク = 半日〜最大2日）

ツリー形式で提示。必要に応じて調整。

**詳細な分解例**: [examples.md](references/examples.md)

## ステップ4: GitHub Projects作成

1. `gh project create` でボードを作成
2. カラムを設定: Backlog / Ready / In Progress / In Review / Done
3. ビューを設定: Board / Table / Roadmap

## ステップ5: Issue作成

1. ラベルを作成（priority: high/medium/low、category: feature/bug/refactor/docs/test）
2. 大タスク用のマイルストーンを作成
3. Issueを作成（以下を含む）:
   - 明確なタイトル
   - 説明（概要、チェックリスト、受け入れ基準、依存関係、参照）
   - ラベル（カテゴリ + 優先度）
   - マイルストーン
4. Projectsボードに追加

## ステップ6: 依存関係セットアップ

1. 依存関係を特定
2. Issue本文に記録（「Dependencies」セクション）
3. 依存するIssueをBacklogに配置 → ブロックが解除されたらReadyに移動

## ステップ7: 進捗管理開始

1. 最初のタスクを特定（依存関係なし + 高優先度）
2. Readyに移動
3. どこから始めるか提案
