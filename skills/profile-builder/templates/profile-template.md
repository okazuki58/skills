# プロフィールテンプレート

選択式の質問に答えた内容から、my-profile/配下のファイルを自動生成するためのテンプレートです。

## SKILL.md テンプレート

```markdown
---
name: my-profile
description: "[氏名]のプロフィール、経歴、スキルセット、現在の業務内容、強み・課題、キャリア目標を管理。人事評価や目標設定時に参照し、最適化された提案を行うためのskill"
---

# [氏名] プロフィール

## 概要

このskillは、[氏名]の詳細なプロフィール情報を管理します。Claude Codeが目標設定、振り返り作成、技術提案などを行う際に、このskillを自動参照することで、状況に最適化された提案が可能になります。

## 使用タイミング

- 人事評価の目標設定時
- 半期・月次・週次の振り返り作成時
- 技術的な提案や学習計画の立案時
- キャリア相談時
- 業務の優先順位づけ時

このskillは**performance-review skill**と連携して使用されます。

## 基本情報

- **氏名**: [氏名]
- **所属**: [所属部署]
- **グレード**: [現在のグレード]（目標: [目標グレード]）
- **エンジニア歴**: [経験年数]
- **年収レンジ**: [年収レンジ]

### 現在のグレードでの実感

[「このグレードの役割、できてるな」と感じるか / 「まだ足りない」と感じるか]

## 現在の役割

**ポジション**: [ポジション説明]

**開発スタイル**: [開発スタイル]

**担当範囲**:
[担当範囲のリスト]

## 詳細情報

詳細は以下のreferenceファイルを参照：

- **経歴・スキルセット**: [background.md](references/background.md)
- **現在の業務・役割**: [current-role.md](references/current-role.md)
- **強み・課題**: [strengths-weaknesses.md](references/strengths-weaknesses.md)
- **キャリア目標**: [career-goals.md](references/career-goals.md)

## 重要な前提

### 前回評価のポイント

[前回評価の要約]

### 今期の重点課題

## Claude Codeへの指示

プロフィール情報に関連するタスク（目標設定、振り返り、技術提案など）を行う際は：

1. このskillを必ず参照する
2. 特に以下を考慮する：
   - エンジニア歴[経験年数]という経験値
   - [開発スタイル]という環境
   - 前回評価での課題
   - [業務内容]という特殊な業務内容
3. references/の詳細情報を適宜参照する
4. performance-review skillと組み合わせて使用する
```
