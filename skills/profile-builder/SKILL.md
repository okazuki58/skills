# プロフィールビルダー

## 概要

このskillは、**10分で完成する簡潔な対話**でエンジニアのプロフィールを作成します。

プロフィールの設計は目標設定の土台となる重要なステップです。

- **10問で完結**: 経験年数・年収・技術理解度チェックで大まかなレベルを把握
- **レベル自動判定**: ジュニア/ミドル/シニアを自動判定し、一般的な課題を提示
- **進捗表示**: 「Q1/10」など、常に進捗を表示
- **所要時間**: 10分程度で完了
- **カスタマイズ可能**: 自動生成後、必要に応じてカスタマイズ

## 使用タイミング

- 初回プロフィール作成時
- 半期ごとのプロフィール更新時
- グレード変更時
- 技術スタック変更時

## レベル自動判定の仕組み

経験年数・年収・グレードから、ジュニア/ミドル/シニアを自動判定し、レベルに応じた一般的な課題・強みを提示します。

### レベル判定基準

| レベル | 経験年数 | 年収レンジ | グレード | 一般的な課題 |
|--------|---------|-----------|---------|------------|
| ジュニア | 1-3年 | 300-500万 | G1-G2 | 基礎知識、コードレビュー、設計力 |
| ミドル | 3-7年 | 500-700万 | G2-G3 | 設計力、チームリード、アウトプット |
| シニア | 7年以上 | 700万以上 | G3-E4 | アーキテクチャ、技術戦略、組織貢献 |

### 技術理解度チェックの仕組み

主要な技術（Ruby、オブジェクト指向、設計パターンなど）について、理解度を数問でチェックし、具体的な課題を特定します。

## プロフィール作成フロー

**所要時間**: 10分程度（10問）
**形式**: 選択式中心（カスタマイズは自由入力）

### 重要な原則

1. **必ず1つずつ質問**: 複数の質問を一度に投げかけない
2. **進捗を明示**: 「Q1/10」など、常に進捗を表示
3. **レベルに応じた自動生成**: 経験年数・年収・グレードから一般的な課題・強みを自動生成
4. **技術理解度チェック**: 主要な技術について数問でチェック
5. **カスタマイズ可能**: 自動生成後、必要に応じて自由入力でカスタマイズ

### 質問リスト（10問）

1. 1年未満
2. 1-3年（ジュニアレベル）
3. 3-5年（ミドルレベル）
4. 5-7年（ミドル〜シニアレベル）
5. 7-10年（シニアレベル）

2. 300-400万（ジュニアレベル）
3. 400-500万（ジュニア〜ミドルレベル）
4. 500-600万（ミドルレベル）
5. 600-700万（ミドルレベル）
6. 700-800万（シニアレベル）
7. 800万以上（シニアレベル）

1. G1 - 見習い（作業について教わりながら完了）

3. G3 - 自走可能（自発的に作業を切り出して進める）

5. E4 - テックリード
6. E5 - VPoE・CTO

#### Q4/10. 主要な技術領域を選んでください（複数選択可）

1. フロントエンド
2. バックエンド
3. インフラ
4. フルスタック（フロント・バック両方）
5. モバイル
6. データエンジニアリング
7. ML/AI

#### Q5/10. 使用している主要な言語・フレームワークを選んでください（複数選択可）

1. Ruby / Rails
2. JavaScript / TypeScript / React / Next.js
3. Python / Django / Flask
4. Go
5. Java / Spring
6. PHP / Laravel
7. その他（自由入力）

#### Q6/10. Rubyの基礎理解度を選んでください

1. 基本的な文法は理解している（変数、条件分岐、ループ）
2. クラス・モジュールを使える
3. ブロック・Proc・lambdaの違いを理解している
4. メタプログラミングができる
5. Rubyは使っていない

#### Q7/10. オブジェクト指向の理解度を選んでください

1. クラスとインスタンスの違いを理解している
2. 継承・ポリモーフィズムを使える
3. SOLID原則を意識して設計できる
4. デザインパターンを実践できる
5. まだ理解していない

#### Q8/10. 設計・アーキテクチャの理解度を選んでください

1. MVC パターンを理解している
2. API 設計ができる
3. DB 設計（正規化・インデックス）ができる
4. マイクロサービス・DDD などの設計手法を実践できる
5. まだ理解していない

#### Q9/10. 現在の課題を選んでください（複数選択可）

**ジュニアレベルの一般的な課題**:
1. 基礎的なプログラミングスキル（文法・アルゴリズム）
2. コードレビューの指摘を理解して修正する力
3. 既存コードを読んで理解する力
4. バグの原因を特定する力
5. 設計書を読んで実装する力

**ミドルレベルの一般的な課題**:
6. 設計・アーキテクチャ（適切な設計ができない）
7. コードレビュー力（的確な指摘ができない）
8. チームリード・メンター経験（教え方がわからない）
9. アウトプット・発信（社内外で発信できていない）
10. 見積もり・タスク管理（工数見積もりがズレる）

**シニアレベルの一般的な課題**:
11. アーキテクチャ設計（大規模システムの設計経験不足）
12. 技術戦略・技術選定（最適な技術選定ができない）
13. 組織貢献（チーム全体の生産性向上に貢献できていない）
14. リーダーシップ（チームを引っ張る力が不足）

15. その他（自由入力）

#### Q10/10. 目指すキャリア方向性を選んでください

1. マネージャー（G4以上）- 人を育てる・チームをリードする
2. スペシャリスト（E4以上）- 技術を深く追求する
3. フルスタックエンジニア - 幅広く対応できる
4. プロダクトマネージャー - プロダクト全体を見る
5. まだ決めていない

### レベル別の自動生成内容

#### ジュニアレベル（経験1-3年、年収300-500万、G1-G2）

**自動生成される課題**:
- Ruby/Rails の基礎知識が不足している
- オブジェクト指向の理解が浅い
- 設計・アーキテクチャの経験がない
- コードレビューで指摘を受けても理解できないことがある
- バグの原因特定に時間がかかる

**自動生成される強み**:
- 学習意欲が高く、新しいことを吸収できる
- 指示通りに実装を進められる
- 素直にフィードバックを受け入れられる

**推奨される目標**:
- Ruby/Rails の基礎を固める（技術書2冊読破）
- オブジェクト指向を理解する（Rubyで学ぶデザインパターン）
- 小規模な設計を経験する
- コードレビューで学ぶ習慣をつける

#### ミドルレベル（経験3-7年、年収500-700万、G2-G3）

**自動生成される課題**:
- 設計力が不足している（設計ミスで作り直すことがある）
- コードレビューの指摘が浅い
- チームリード・メンター経験が少ない
- 社内外でのアウトプット・発信ができていない
- 見積もりがズレることがある

**自動生成される強み**:
- 基本的な実装は問題なくできる
- バグ修正ができる
- コードレビューができる（指摘の質を上げたい）
- 新しい技術をキャッチアップできる

**推奨される目標**:
- 設計力を向上させる（設計関連の技術書を読む）
- コードレビュー力を強化する（レビューチェックリスト作成）
- チームメンバーにメンターとして教える経験を積む
- 技術ブログ・社内発信を始める（月1本）
- 見積もり精度を上げる（タスク分解の練習）

#### シニアレベル（経験7年以上、年収700万以上、G3-E4）

**自動生成される課題**:
- 大規模システムのアーキテクチャ設計経験が不足
- 技術戦略・技術選定の経験が少ない
- チーム全体の生産性向上に貢献できていない
- リーダーシップ・組織づくりの経験が不足

**自動生成される強み**:
- 設計・実装・レビューは一通りできる
- チームメンバーに教えられる
- プロジェクト全体を見られる
- 技術的な判断ができる

**推奨される目標**:
- アーキテクチャ設計を学ぶ（DDD, マイクロサービスなど）
- 技術選定の経験を積む（新技術導入のリード）
- チーム全体の生産性向上施策を実施（CI/CD改善、開発プロセス改善）
- 技術発信を継続する（外部登壇、技術ブログ）
- リーダーシップを発揮する（プロジェクトリード、メンバー育成）

## 出力ファイル生成

すべての質問に答えたら、レベルに応じたプロフィールを自動生成します。

### 生成ファイル

```
my-profile/
├── SKILL.md                      # メインプロフィール
└── references/
    ├── background.md             # 経歴・スキルセット
    ├── current-role.md           # 現在の業務・役割
    ├── strengths-weaknesses.md   # 強み・課題
    └── career-goals.md           # キャリア目標
```

### 生成ロジック

1. **レベル判定**: 経験年数・年収・グレードからジュニア/ミドル/シニアを判定
2. **技術理解度の把握**: Ruby基礎・オブジェクト指向・設計の理解度をチェック
3. **課題の特定**: レベルに応じた一般的な課題 + ユーザーが選択した課題
4. **強みの特定**: レベルに応じた一般的な強み
5. **目標の提案**: レベルに応じた推奨目標

### 各ファイルへのマッピング

| ファイル | 含まれる情報 |
|---------|-------------|
| `SKILL.md` | レベル、グレード、経験年数、年収レンジ、主要技術、キャリア方向性 |
| `background.md` | 技術スタック、使用言語・フレームワーク、技術理解度（Ruby基礎、オブジェクト指向、設計） |
| `current-role.md` | 現在の役割（レベルに応じた標準的な役割を記載） |
| `strengths-weaknesses.md` | 強み（レベル別の一般的な強み）、課題（レベル別の一般的な課題 + ユーザー選択）|
| `career-goals.md` | 目指す方向性、レベルに応じた推奨目標 |

## 重要な設計原則

1. **10分で完結**: 質問数を10問に抑え、短時間で完了

5. **カスタマイズ可能**: 自動生成後、必要に応じて編集可能

## goal-management skillとの連携

プロフィール作成完了後、goal-management skillが自動的に`my-profile/`を参照し：

1. レベル・経験年数に基づいた目標提案
2. 課題を反映した半期目標生成
3. 技術スタックに基づいた学習計画提案
4. キャリア目標に沿った成長プラン作成

を行います。

## Claude Codeへの指示

このskillを実行する際は、以下を厳守してください：

### 必須事項

1. **開始時に所要時間を提示**: 「10分程度（全10問）」と明示
2. **必ず一問一答形式**: 質問は1つずつ。複数の質問を同時に投げかけない
3. **進捗を明示**: 「Q1/10」など、毎回進捗を表示
4. **レベル自動判定**: 経験年数・年収・グレードからジュニア/ミドル/シニアを自動判定
5. **一般的な課題を自動生成**: レベルに応じた一般的な課題を提示
6. **技術理解度チェック**: Ruby基礎・オブジェクト指向・設計について質問
7. **ユーザーの回答を待つ**: 回答がない状態で次の質問に進まない

### 既存プロフィールの確認方法

**重要**: `my-profile/`はプロジェクトルートに作成されます。`.claude/skills/`配下ではありません。

既存プロフィールを確認する際は：

```
プロジェクトルート/my-profile/SKILL.md
```

を探してください。`.claude/skills/my-profile/`ではありません。
