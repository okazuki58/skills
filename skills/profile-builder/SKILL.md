# プロフィールビルダー

クイックアセスメントで約10分で簡潔なエンジニアプロフィールを作成。経験/年収/技術理解度に基づいて課題を自動生成。プロジェクトルートの `my-profile/` に出力。

**主な機能**:
- 全10問（選択式）
- レベル自動判定（ジュニア/ミドル/シニア）
- レベル別の一般的な課題を自動生成
- 10分で完了
- 生成後にカスタマイズ可能

## レベル自動判定

| レベル | 経験年数 | 年収レンジ | グレード | よくある課題 |
|-------|---------|-----------|---------|-------------|
| ジュニア | 1-3年 | 300-500万 | G1-G2 | 基礎、コードレビュー、設計 |
| ミドル | 3-7年 | 500-700万 | G2-G3 | 設計、チームリード、アウトプット |
| シニア | 7年以上 | 700万以上 | G3-E4 | アーキテクチャ、戦略、組織貢献 |

## ワークフロー

**所要時間**: 約10分（10問）
**形式**: 選択式（カスタマイズ可能）

### 重要な原則

1. **1つずつ質問**: 複数の質問を束ねない
2. **進捗を表示**: 常に「Q1/10」「Q2/10」などを表示
3. **レベル別自動生成**: 経験/年収/グレードに基づいて一般的な課題/強みを生成
4. **技術理解度チェック**: 主要技術の理解度を確認
5. **カスタマイズ可能**: 自動生成後に自由形式でのカスタマイズを許可

### 質問フロー

**Q1/10**: エンジニア歴（1年未満 / 1-3年 / 3-5年 / 5-7年 / 7-10年 / 10年以上）

**Q2/10**: 現在の年収レンジ（300万未満 / 300-400万 / 400-500万 / 500-600万 / 600-700万 / 700-800万 / 800万以上）

**Q3/10**: 現在のグレード（G1 / G2 / G3 / G4 / E4 / E5）

**Q4/10**: 主な技術領域（複数選択: フロントエンド / バックエンド / インフラ / フルスタック / モバイル / データ / ML-AI）

**Q5/10**: 主な言語・フレームワーク（複数選択: Ruby-Rails / JS-TS-React-Next / Python / Go / Java / PHP / その他）

**Q6/10**: Ruby理解度（基本 / クラス・モジュール / ブロック・Proc・lambda / メタプログラミング / 使っていない）

**Q7/10**: オブジェクト指向理解度（クラス・インスタンス / 継承・ポリモーフィズム / SOLID / デザインパターン / まだ）

**Q8/10**: 設計・アーキテクチャ理解度（MVC / API設計 / DB設計 / マイクロサービス・DDD / まだ）

**Q9/10**: 現在の課題（複数選択: レベルによって異なる - [questions.md](references/questions.md)参照）

**Q10/10**: キャリア方向性（マネージャー / スペシャリスト / フルスタック / プロダクトマネージャー / 未定）

**詳細な質問**: [questions.md](references/questions.md)

## レベル別自動生成コンテンツ

### ジュニア（1-3年、300-500万、G1-G2）

**自動生成される課題**:
- Ruby/Rails の基礎知識が不足
- オブジェクト指向の理解が浅い
- 設計・アーキテクチャの経験がない
- コードレビューのフィードバック理解が困難
- バグの特定が遅い

**自動生成される強み**:
- 高い学習意欲
- 指示通りに実装できる
- フィードバックを受け入れる

**推奨される目標**:
- Ruby/Railsの基礎を固める（技術書2冊読破）
- オブジェクト指向を理解（Rubyデザインパターン）
- 小規模設計を経験
- コードレビューから学ぶ

### ミドル（3-7年、500-700万、G2-G3）

**自動生成される課題**:
- 設計力不足（設計ミスで作り直し）
- コードレビューの指摘が浅い
- チームリード・メンター経験が限定的
- 社内外でのアウトプット・発信がない
- 見積もりのミス

**自動生成される強み**:
- 基本的な実装は可能
- バグ修正ができる
- コードレビューができる（質を向上させたい）
- 新技術をキャッチアップできる

**推奨される目標**:
- 設計力を向上（設計関連の技術書を読む）
- コードレビュー力を強化（チェックリスト作成）
- メンター経験を積む
- 技術ブログ・社内発信を開始（月1本）
- 見積もり精度を向上（タスク分解の練習）

### シニア（7年以上、700万以上、G3-E4）

**自動生成される課題**:
- 大規模システムアーキテクチャ設計経験が限定的
- 技術戦略・技術選定経験が限定的
- チーム全体の生産性向上に貢献していない
- リーダーシップ・組織構築経験が限定的

**自動生成される強み**:
- 設計・実装・レビューを一通り扱える
- チームメンバーに教えられる
- プロジェクト全体を見られる
- 技術的な判断ができる

**推奨される目標**:
- アーキテクチャ設計を学ぶ（DDD、マイクロサービスなど）
- 技術選定経験を積む（新技術導入のリード）
- チーム生産性向上施策を実行（CI/CD、プロセス改善）
- 技術発信を継続（外部登壇、技術ブログ）
- リーダーシップを発揮（プロジェクトリード、メンバー育成）

## 出力ファイル生成

全質問の後、レベルに応じてプロフィールを自動生成。

### 生成されるファイル

```
my-profile/
├── SKILL.md                      # メインプロフィール
└── references/
    ├── background.md             # 経歴、スキルセット
    ├── current-role.md           # 現在の業務、役割
    ├── strengths-weaknesses.md   # 強み、課題
    └── career-goals.md           # キャリア目標
```

### 生成ロジック

1. **レベル判定**: 経験/年収/グレードからジュニア/ミドル/シニアを判定
2. **技術理解度**: Ruby基礎/オブジェクト指向/設計理解度をチェック
3. **課題特定**: レベルベースの一般的な課題 + ユーザー選択
4. **強み特定**: レベルベースの一般的な強み
5. **目標提案**: レベルベースの推奨目標

**重要**: プロジェクトルートの `my-profile/` に保存。`.claude/skills/profile-builder/my-profile/` ではない。

## goal-management との連携

プロフィール作成後、goal-management が `my-profile/` を自動参照:
1. レベル/経験に基づいて目標を提案
2. 課題を反映した半期目標を生成
3. 技術スタックに基づいて学習計画を提案
4. キャリア目標に沿った成長計画を作成
