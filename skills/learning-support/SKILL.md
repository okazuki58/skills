# 学習サポート

## ワークフロー

1. 理解度チェック
2. 学習計画立案（5-7ステップ）
3. リポジトリ作成
4. GitHub Projectsセットアップ
5. Issue作成
6. ハンズオン学習サポート（7ステップサイクル）
7. 完了サマリー → Issueコメント → クローズ

## ステップ1: 理解度チェック

AskUserQuestionで以下を確認:
- 現在の理解レベル（初心者/中級者/上級者）
- 学習目的（基礎/実践/資格取得/深い理解）
- タイムライン（1週間/2週間/1ヶ月/2-3ヶ月/柔軟）

## ステップ2: 学習計画立案

チェック結果に基づいてトピックを5-7ステップに分解。各ステップ = 1つの概念（密接に関連する場合は最大2つ）。

各ステップに以下を定義:
- ステップ名（「Day 1: Xの基礎」）
- 学習目標
- 成果物
- 推定時間

計画を提示し、必要に応じて調整。

**詳細な例**: [examples.md](references/examples.md)

## ステップ3: リポジトリ作成

1. リポジトリ名 `{topic}-learning` を提案
2. AskUserQuestionで公開範囲を確認
3. `gh repo create` でリポジトリ作成
4. 学習計画を含む README.md を作成

## ステップ4: GitHub Projectsセットアップ

1. `gh project create` でProjectsボードを作成
2. カラムを設定: Not Started / In Progress / Completed

## ステップ5: Issue作成

1. 各ステップのIssueを作成
2. Projectsボードに追加
3. READMEにIssue番号を更新

## ステップ6: ハンズオン学習サポート

**トリガー**: 「Day X を学習したい」/「Day X を復習したい」/「Issue #X を学習したい」

**プロセス**:
1. `gh issue view` でIssueを読み込み
2. 5-7個のサブステップに分解
3. 各サブステップで7ステップ学習サイクルを実行:

### 7ステップ学習サイクル

1. **概念説明** - 最大5文、1-2個のシンプルな例
2. **小さな演習** - 焦点を絞ったタスク、5-10分
3. **ユーザー実装** - コードを待つ、求められない限りヒントは出さない
4. **コードレビュー** - 良い点、改善点、理解度確認の質問
5. **解答 + 「なぜ」** - 解答例、理由の説明、実世界との接続
6. **良いコード vs 悪いコード** - 2-3個の対比、違いの説明
7. **理解度確認** - 「理解できましたか？」 → はい: 次へ / いいえ: 再説明

**重要**:
- ステップをスキップしない
- 複数の概念を組み合わせない
- 理解せずに進まない
- 常に「なぜ」を説明
- 説明は日本語、技術用語は初回は英語+日本語
- 可能な場合は my-profile を参照してカスタマイズ

## ステップ7: 学習完了

**トリガー**: 「終わり」/「完了」/「終了」

1. サマリーを生成（概念、コード、重要ポイント、ベストプラクティス、次のステップ、チャットリンク）
2. `gh issue comment` でIssueに投稿
3. `gh issue close` でIssueをクローズ
4. Projectsボードを更新
